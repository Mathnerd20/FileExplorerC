# Prerequisites for Building a CLI File Explorer in C (Linux)

To implement your file explorer using a Left-child, Right-sibling (LCRS) tree structure, you should study the following areas:

## 1. Directory Handling (<dirent.h>)
*   **opendir(const char *name):** Opens a directory stream.
*   **readdir(DIR *dirp):** Returns a pointer to a `struct dirent` representing the next directory entry.
*   **struct dirent:** Contains `d_name` (filename) and `d_type` (entry type).
*   **closedir(DIR *dirp):** Closes the directory stream.

## 2. File Metadata (<sys/stat.h>)
*   **stat(const char *path, struct stat *buf):** Retrieves detailed information about a file.
*   **struct stat:** Study fields like `st_size` (size), `st_mode` (type/permissions), and `st_mtime` (last modification).
*   **Type Macros:** Use `S_ISDIR(st_mode)` and `S_ISREG(st_mode)` to distinguish folders from files.

## 3. Path Management
*   **Absolute vs. Relative Paths:** Understanding how to navigate from your Root/Home.
*   **String Manipulation:** Use `snprintf` or `strcat` to build paths (e.g., "parent/folder" + "/" + "filename").
*   **PATH_MAX:** Defined in `<limits.h>`, use this for buffer sizes instead of fixed small numbers like 20.

## 4. Tree Serialization & Persistence
*   **Serialization:** How to save a pointer-based structure to a flat file.
*   **Tree Traversal:** Pre-order traversal is typically used to save and reconstruct tree hierarchies.
*   **File I/O:** Practice using `fread` and `fwrite` for binary data, or structured text formats.

## 5. Memory Management
*   **Recursive Freeing:** Ensuring that when a directory node is deleted, all children and siblings are properly `free()`'d.
*   **Dynamic Allocation:** Using `malloc` for filenames to handle varying lengths safely.

## 6. CLI Interaction
*   **fgets() vs scanf():** Use `fgets` to read user commands to handle spaces in filenames and prevent buffer overflows.
*   **errno.h:** Use `perror()` to debug why a file operation (like opening a restricted folder) failed.
*   **ncurses (Optional):** For building a TUI (Text User Interface) with arrow-key navigation.
